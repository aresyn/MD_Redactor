# MD Redactor

[English](README.md) | Русский

MD Redactor помогает готовить Markdown-тексты к редактуре человеком или ИИ-агентом. Вы открываете `.md` файл как обычный отформатированный документ, выделяете фрагмент, нажимаете `Enter` и пишете комментарий справа. Комментарий сохраняется прямо в Markdown рядом с фрагментом.

Главная идея простая: документ остается одним файлом. Без базы данных, sidecar-json и отдельного списка замечаний. Такой файл можно отправить автору, редактору или ИИ-агенту, и все правки останутся внутри текста.

## Для чего это нужно

- Отмечать точные места в прозе, статье или сценарии, не копируя фрагменты в отдельный документ.
- Писать комментарии рядом с текстом, к которому они относятся.
- Работать с Markdown как с документом: заголовки, списки, цитаты и выделения видны сразу.
- Передавать файл дальше без потери контекста.
- Не бояться удаления правок: существующие номера не меняются.
- Открывать русские Markdown-файлы в UTF-8 и старые файлы Windows-1251 без порчи кириллицы.

## Что уже умеет MVP

- Нативное Windows-приложение для Windows 10+.
- WYSIWYG-редактор Markdown на WebView2 и ProseMirror.
- Светлая, темная и системная тема.
- Русский и английский интерфейс. По умолчанию используется язык Windows.
- Открытие `.md` через кнопку, `Ctrl+O`, drag-and-drop или Windows «Открыть с помощью...».
- Создание правки по сценарию: выделить текст, нажать `Enter`, написать комментарий.
- Правая панель правок: список, активная правка, переход к фрагменту, редактирование комментариев и удаление правки.
- Безопасное сохранение: проверка тегов, атомарная запись и `.bak` перед первой записью в сессии.
- Поддержка UTF-8, UTF-8 BOM, UTF-16 LE/BE BOM и Windows-1251 fallback.

## Установка

Соберите или скачайте установщик:

```text
artifacts\installer\MDRedactorSetup-x64.exe
```

Запустите его. Приложение ставится для текущего пользователя без прав администратора:

```text
%LOCALAPPDATA%\Programs\MD Redactor
```

Инсталлятор добавляет ярлыки на рабочий стол и в меню Пуск. MD Redactor также появляется в Windows «Открыть с помощью...» для `.md` файлов, но не назначает себя редактором Markdown по умолчанию.

Release-сборка включает .NET runtime для Windows x64. WebView2 Runtime должен быть установлен на компьютере.

## Основной сценарий

1. Откройте Markdown-файл.
2. Выделите фрагмент в отформатированном документе.
3. Нажмите `Enter`.
4. Напишите комментарий в карточке справа.
5. Сохраните файл через `Ctrl+S` или кнопку `Сохранить`.
6. Передайте этот же Markdown-файл ИИ-агенту или другому редактору.

Если выделение пересекается с уже существующей правкой, новая правка не создается. Приложение активирует существующую правку и показывает уведомление.

## Формат правок

Блочная правка:

```markdown
<!-- ed-start id="1" -->
фрагмент
<!-- ed-comm id="1"
комментарий
-->
<!-- ed-end id="1" -->
```

Inline-правка:

```markdown
Текст до <!-- ed-start id="2" -->фрагмент<!-- ed-comm id="2"
комментарий
--><!-- ed-end id="2" --> текст после.
```

Все три маркера одной правки используют одинаковый положительный `id`. Номера не обязаны идти подряд. Если из набора `#1`, `#2`, `#5` удалить `#2`, останутся `#1` и `#5`. Новая правка получит `max(existing id) + 1`.

У правок нет статусов. Атрибуты `status`, `resolved`, `open` и похожие состояния не входят в формат.

## Сохранение и резервные копии

Перед записью MD Redactor проверяет служебные теги. Сохранение блокируется, если найдены дубли id, разные id внутри одной правки, незакрытые маркеры, неподдерживаемый `status` или опасные последовательности для HTML comments.

Сохранение устроено так:

1. Рядом с Markdown создается временный файл.
2. Перед первой записью в текущей сессии создается backup `filename.md.bak`.
3. Исходный файл заменяется временным.
4. Если происходит ошибка, исходный файл остается нетронутым.

Если файл был изменен другой программой после открытия, приложение спросит, перезаписывать ли его.

Ошибки чтения и сохранения пишутся в:

```text
%LOCALAPPDATA%\MDRedactor\logs
```

В лог попадает тип ошибки, контекст и путь файла. Текст документа туда не записывается.

## Кодировки

Внутри приложения текст хранится как Unicode. При чтении распознаются:

- UTF-8 BOM;
- UTF-8 без BOM;
- UTF-16 LE/BE по BOM;
- Windows-1251 как fallback, если файл невалиден как UTF-8.

При сохранении открытого файла приложение старается сохранить исходную кодировку. Новые файлы и неизвестные кодировки сохраняются как UTF-8 без BOM.

## Горячие клавиши

- `Ctrl+O`: открыть Markdown-файл.
- `Ctrl+S`: сохранить текущий файл.
- `Enter` при выделении в редакторе: создать правку.
- `Enter` в поле комментария: добавить перенос строки.
- `Escape` в редакторе: снять активную правку.

## Сборка из исходников

Нужны:

- Windows 10 или новее;
- .NET 10 SDK;
- Node.js и npm;
- WebView2 Runtime.

Полная сборка:

```powershell
.\scripts\build.ps1
```

Скрипт проверяет окружение, собирает `web/editor`, запускает web-тесты, затем выполняет restore, build и test для .NET solution.

Запуск из исходников:

```powershell
dotnet run --project .\src\MDRedactor.App\MDRedactor.App.csproj
```

Перед запуском соберите web-редактор через `.\scripts\build.ps1` или выполните `npm run build` в `web\editor`.

## Упаковка и инсталлятор

Создать release-папку:

```powershell
.\scripts\package.ps1
```

Результат:

```text
artifacts\publish\win-x64
```

Собрать Windows-инсталлятор:

```powershell
.\scripts\installer.ps1
```

Результат:

```text
artifacts\installer\MDRedactorSetup-x64.exe
```

Инсталлятор собирается через Inno Setup и поддерживает русский и английский язык.
